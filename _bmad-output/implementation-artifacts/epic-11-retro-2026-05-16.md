## 2. What Went Well

- **Adversariales Review-Architektur skaliert.** Three parallel subagents (Blind Hunter / Edge Case Hunter / Acceptance Auditor) over a single diff file (`_e11-review-diff.tmp`, 2772 lines, committed + uncommitted + untracked) delivered **orthogonal** findings with minimal overlap. Edge Case Hunter found IPv6/userinfo/path-injection in the parser, Blind Hunter found BetaBanner case-sensitivity, Acceptance Auditor verified ADR-015/017/020 compliance + REQ-088 AC-4 status. Evidence: of 8 patches applied, none collided with another patch's scope.
- **A24-influenced patch depth (no half-measures).** Although no patches were labeled `[Review][Patch][Interim]` (none qualified), the **mindset** from E10-A24 carried over: every BetaBanner patch (case-insensitive, lazy useState, try/catch, fork-friendly fallback) was conceived as "deeper form" rather than minimum-viable. The mid-test `||` vs `??` correction (Node's `process.env` stringification of `undefined`) shows defensive-by-default thinking.
- **Story files as single-source-of-truth held.** E11-S3 was created from a one-line stub to 360+ lines of dev-ready context via `bmad-create-story` (incl. an 8-item Previous-Story-Intelligence section pulling E11-S2 learnings). The dev-agent then made exactly **one** deliberate scope expansion (`outputFileTracingRoot`) — transparent in Completion Notes. Auditor verdict: "AC-4 DEVIATION but documented + justified". Textbook "dev-agent owns the system working end-to-end" behavior.
- **Hybrid workflow (DS per story, CR+ER per epic) surfaced cross-story issues.** Examples per-story reviews would have missed:
  - **D4 — Orphan Hangfire schedule on flag flip** (`true → false`): only visible at epic boundary with cross-story sight (S1 introduced env var, S2 wired flag, but no story considered the flip transition).
  - **P5 — `NEXT_PUBLIC_FEEDBACK_URL` undocumented in `.env.example`**: consumed in E11-S2 BetaBanner, claimed by Dev Notes "this story should append it", missed by per-story implementation.
  - **AC-4 DEVIATION (`outputFileTracingRoot`)**: contextual "DEVIATION but documented" verdict only meaningful at epic boundary where the AC-4 enforcement vs AC-2 functional requirement could be weighed.
- **`AskUserQuestion`-driven decision resolution stayed clean.** All 4 review decisions (D1 rustfsadmin handling / D2 AC-9 test coverage gap / D3 BetaBanner case-sensitivity / D4 orphan-Hangfire schedule) presented as one batched call with 2-3 concrete options + tradeoffs. Resolution took one user message. [feedback_decisions_via_ask_tool](memory/feedback_decisions_via_ask_tool.md) pattern paid off again.
- **Beta-vs-Production parity verified empirically.** All 7 `IsDevelopment()` sites audit-confirmed clean (no `|| envName == "Beta"` relaxations). ADR-015 invariant intact. The audit was **verification**, not coding work — meaning the architectural intent shipped in E11-S2 was already correct.

## 3. What Didn't Go Well — Patterns

*(Systemic observations, no blame. Each links to an action item.)*

- **AC-2 partial blieb partial — eager-init Blocker nicht antizipiert.** E11-S2 spec named `rustfsadmin` cleanup "top removal priority" for REQ-088 AC-4. Dev-agent hit the Hangfire `PostgreSqlStorage..ctor` + `Get<DocumentStorageSettings>()` singleton-closure eager-init blocker, documented it honestly, deferred 5 of 7 keys. **But:** a spike before story-start would have surfaced the blocker earlier and either expanded scope OR explicitly planned the partial state. The "low-risk mechanical" spec language overstated certainty. → **A28** (Spike-First Pattern for "low-risk mechanical" cleanup specs)
- **AC-9 test coverage gap — Spec promised 5 hardening assertions, delivered 1.** `BetaEnvironmentHardeningTests` covers `RequireHttpsMetadata` only; Swagger 404 / Hangfire 404 / HSTS / HTTPS-redirect / strict-CORS deferred to E15-S2 unblocker. Surfaced at epic-review (D2), should have surfaced at story-review. Pattern: **ACs with `N of M` sub-points need explicit "all subitems covered" validation at story close**. → **A29** (AC-Subitem Completion Check)
- **Task 11 "Manual smoke tests" marked `[x]` despite dev-agent non-interactivity.** Process smell, not E11-specific — pattern across all stories with manual-verification ACs. Dev-agent documented "expected outcomes" instead of "observed evidence". A green checkbox in Tasks/Subtasks visually claims dev-completion, but the work is actually pending-human-verification. → **A30** (Three-State Task Status for Manual ACs)
- **rustfsadmin defer is deployment-gating, not just background tech-debt.** Unlike E10's audit-actor-identity defer (cross-cutting, long timeline), the rustfsadmin defer **gates any actual Beta Railway deployment** because images would bundle the literal credential. Currently tracked in `deferred-work.md` but easy to lose track of when E12-S1 starts. → addressed implicitly through E12-S1 scope-planning (see Significant Discoveries §9).
