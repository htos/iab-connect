## What to improve (→ action items)

- **The "reset form state after firing the mutation" trap.** Three Med error-path regressions (P1/P2/P3) all shared one root cause: a god-page that cleared/closed form state ONLY in its success branch (after `await`) was migrated to a fire-and-forget `mutate()` followed by a SYNCHRONOUS reset — silently wiping the user's input on the error path. The outcome-level characterization net didn't catch it. → **A92.**
- **The deterministic-404 retry, masked by the test harness.** `use-board-document` + `use-profile` throw a not-found sentinel on 404, which the provider `retry:1` default retried (double-fetch + ~1s delay). The harness `retry:false` masked it — invisible until review. → **A93.** (Latent in earlier slices' 404 sentinels too.)
- **Wrap-vs-rewrite as an explicit create-story decision.** The cleanest net-survival outcome (documents) came from wrapping an existing lib module; the costlier one (profile transport adaptation) came from a raw-fetch page with no module. Making "does a lib module already own these URLs + does a sibling share it → WRAP" an explicit DEC-1 default would have pre-decided S2/S3 with zero spec churn. → **A94.**

## Action items (promoted to project-context.md)

- **A92** — Migrating an await-then-reset god-page handler (dialog/inline-editor that cleared or closed ONLY on success) to a fire-and-forget TanStack mutation: the reset/close MUST be driven by the mutation outcome (onSuccess / mount-on-open / parent-controlled), NEVER run synchronously after `mutate()` — else the error path silently wipes the user's input (dialog stays open but empty; editor collapses losing edits). Add an "input preserved on error" assertion for any form-in-a-dialog/inline-editor; the outcome-level net (service-fn fired + error surfaced) does NOT catch it. Refines A76/A79.
- **A93** — A query whose `queryFn` throws a deterministic not-found sentinel on 404 (`BoardDocumentNotFoundError`/`NoMemberRecordError`/`*NotFoundError`) MUST disable retry for that sentinel: `retry: (n, err) => !(err instanceof Sentinel) && n < 1`. The provider default `retry:1` otherwise double-fetches + delays the not-found/no-member view ~1s vs the god-page's immediate render. The characterization harness's `retry:false` MASKS the prod `retry:1`, so this is invisible until boundary review — audit every 404-sentinel query in the program (E29 fixed board-documents + profile; events/members/suppliers/sponsors `*NotFoundError` queries likely carry the same latent delay). Specializes A79.
- **A94** — Wrap-vs-rewrite at create-story time: when a god-page ALREADY routes through a `lib/services`/`lib/api` module, the slice `api` layer should WRAP that module (+ a feature-local query-key factory) rather than re-implement the URLs via `useApiClient`. Wrapping keeps the characterization net's transport mock intercepting (the net survives with ZERO transport edits — see documents S2, unchanged spec) AND protects any sibling slice sharing the module (A62). Only raw-`fetch` god-pages (no module) need the E24-S2-style transport adaptation (mock the client, assert `apiClient.get/put`). Make this DEC-1's default branch; decide by "does a lib module already own these URLs + does a sibling share it". Generalizes A62/A84.

A28-A91 carry forward.
