## What was bumpy

- **The A72 prettier-drift trap (mid-S2).** An unreliable drift probe (`git show HEAD:file > /tmp/probe`) gave a false "clean" because `/tmp` is outside the repo (Prettier used default config, no tailwind plugin) and the cwd-relative `git show` path was wrong. `prettier --write` then ballooned a 1-line import change into a 535-line whole-file reformat. Recovered by revert + hand-applying the import only. → **A81**.
- **The half-migrated pilot was invisible.** The E21 Suppliers remainder (detail/new/edit god-pages) existed only as an "Open tech debt" sentence in `architecture-frontend.md` — not a tracked story. Harry found it by opening a file, not via sprint-status. → **A82**.
- **A76 was applied to the list but not symmetrically to every delete surface.** The detail delete (same shared dialog) went unpinned until review. → **A80**.

## Action items promoted to project-context.md (A80-A82)

- **A80 (refines A76)** — A76 coverage is PER destructive surface, not just the canonical one. When the same destructive action appears on a list row AND a detail header, the affordance + failure-branch assertions must cover each surface; a green suite on one does not protect the other even when they share the dialog component.
- **A81 (refines A72/A73)** — the authoritative "is this file prettier-drifted at HEAD?" probe must write the HEAD blob to a temp file INSIDE the repo (so `.prettierrc` + the tailwind plugin resolve) and verify the `git show` produced non-empty content; a `/tmp` probe or a wrong-path `git show` falsely reports "clean" and `prettier --write` then balloons a pre-drifted file.
- **A82** — a pilot/foundation epic's deliberately-deferred remainder must be tracked as an explicit `ready-for-dev` story (or a sprint-status backlog item naming the feature), not just a prose tech-debt note, so the half-migration is visible in the tracker and not discovered by chance.
