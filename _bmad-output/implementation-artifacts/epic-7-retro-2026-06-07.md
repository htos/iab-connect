## What went wrong (→ action items)

1. **Self-inflicted HIGH at review: `prettier --write` on pre-drifted modified files produced ~2000 lines of unrelated reformatting** + CRLF churn on 2 `.cs` files; the initial epic diff was 2554/1193 vs the real 284/55. Root cause: misapplying A58 — the changed-files gate is a *check*, but `prettier --write` rewrites the whole (pre-drifted) file. Compounded by an unreliable drift-check (piping the HEAD blob through git-bash flipped line endings → false "clean"). Found and fully remediated in the boundary review (restore HEAD + re-apply logical changes only). → **A72, A73**.
2. **Frontend render-test coverage was thinner than the backend.** E7-S4's event-form *submit* path is `[!]` manual-verify (heavy page-mock cost) while the public badge got a real render test; E7-S1/S2's runtime a11y (keyboard, focus rendering, contrast) is necessarily `[!]`. Acceptable per A47 but worth noting the asymmetry. No new action (A47 already governs).

## Action items promoted to project-context.md

- **A72** — A58's changed-files gate is **check-only**; never `prettier --write` a pre-drifted *modified* file (it reformats the whole file and bundles an unrelated cleanup into the feature epic). Format only genuinely new files; for modified pre-drifted files hand-match the surrounding style and rely on the eslint changed-files gate + a final `prettier --check` (not `--write`).
- **A73** — On a `core.autocrlf=false` repo whose index is LF, keep edited files LF (Edit/Write tooling can introduce CRLF on Windows → whole-file `git diff` churn). Verify with `git ls-files --eol`; for an authoritative prettier-drift check write the HEAD blob to a temp file via a binary-safe redirect, never a shell pipe (pipes can flip EOL and give a false "clean").

A28–A71 carry forward.
