## What didn't go well

1. **Serilog single-logger constraint forced too many test pivots.** The `Log.Logger = ...` static-frozen-after-Program-Main pattern in `Program.cs:8-11` means we cannot instantiate a second `WebApplicationFactory<Program>` with a different `EnvironmentName`. This blocked:
   - E14-S2: HSTS + HTTPS-redirect runtime tests → code-audit pivot
   - E14-S3: Beta-env-derived factory → transitive Testing-env coverage argument (acceptable but weaker)
   - E14-S4: rate-limiter named-policy + actual 429 transition tests
   Each pivot is individually defensible. Aggregate cost is 6 ACs partially-met-via-loophole. **The proper fix is a `Program.cs` refactor** that makes Serilog setup idempotent across WebApplicationFactory instances. Tracked as a follow-up.
2. **DEC-1 spec divergence in E14-S4** (`/api/v1/auth/*` does not exist in this codebase). The refresh surfaced the gap; DEC-1=A translated to IdentityEndpoints + UserEndpoints; the SCP-2026-05-15 wording assumed a Keycloak-internal auth route. Recording the translation in story files is correct but doesn't prevent future SCPs from making the same assumption.
3. **The audit script's pwsh-only runtime guard blocked dev-time self-test on the local box.** Harry's machine doesn't have pwsh installed (winget install was interrupted). The inline PowerShell 5.1 dry-run via `git grep` + the script's allowlist logic was byte-equivalent but bypassed AC-11's own runtime guard. Q1-Q3 deferred per A47 (Harry runs `pwsh ./scripts/audit-secrets.ps1` after install).
4. **Section 21 ordering bug** during runbook authoring (Section 21 inserted before Section 20 instead of after). Caught + fixed via two-step Edit, but cost 3 tool-calls. Pattern: when authoring back-to-back doc-bundle sections via Edit `old_string` + `new_string`, the "anchor before insertion point" can be the SAME anchor for two consecutive sections — needs `replace_all=false` discipline + grep-verify after each insertion.
5. **Some Acceptance Auditor findings were AC-text quality issues.** E14-S5 AC-1 was internally inconsistent (asked for JSON `Destructure` array, then said JSON cannot express predicates, then said use code-registration). The story's File List correctly recorded the actually-shipped approach. **Action**: AC-text reviewers (PRD validation or sprint-planning) should catch these internal inconsistencies.

## Action items

### A49 — Refactor Program.cs Serilog setup to be WebApplicationFactory-idempotent
**Status**: deferred-work item proposed
**Source**: Epic-14 retro (recurring constraint across E14-S2, E14-S3, E14-S4)
**Description**: The current `Log.Logger = new LoggerConfiguration()...CreateBootstrapLogger()` at Program.cs:8-11 + `try { ... } finally { Log.CloseAndFlush(); }` at lines 13-147 means a second WebApplicationFactory instance in the same process trips "logger is already frozen". This forced 3 stories to pivot from runtime tests to code-audit regex tests, leaving 6 ACs partially-met-via-loophole.
**Fix**: Wrap Serilog setup in a re-entrancy-safe pattern (e.g., `Log.Logger ??= ...` or a static lazy initializer that skips on re-entry, or a `ProgramFixture` test pattern that resets Log.Logger). When fixed, retroactively add the runtime tests for HSTS max-age value, HTTPS 307 + Location, 429 transition + Retry-After + CorrelationId propagation.
**Impact**: enables ~5 deferred runtime tests; closes the AC-text-vs-shipped-test loopholes.

### A50 — Tighten audit-secrets.ps1 allowlist matching to anchored patterns (Edge-10 follow-up)
**Status**: deferred-work item proposed
**Source**: Edge Case Hunter — Epic-14 boundary review
**Description**: Current `$StringAllowlist` uses `$Line.Contains($key)` substring match. A real secret line containing an allowlist substring as prefix or suffix would silently pass. E.g., `password: dev-secret-change-me-9f3a-PROD-CREDENTIAL` passes because it contains `dev-secret-change-me`.
**Fix**: Convert each `$StringAllowlist` key from substring to anchored shape — either regex with start/end anchors OR equality check against the line's value-portion (parse `key: value` and equate `value`). Run the modified script against current tree; verify AUDIT_OK still holds.
**Impact**: prevents allowlist-bypass via real-secret-with-allowlisted-prefix.

### A51 — A31 cross-story orthogonal-AC pattern formalize
**Status**: project-context.md candidate addition
**Source**: Epic-14 retro + E14-S5 AC-4 implementation
**Description**: E14-S5's `AllowlistParity_DestructureFieldsCoverAuditScriptAllowlist` test reads `scripts/audit-secrets.ps1` and asserts the destructure HashSet covers the script's grep patterns. **Promote to project-context.md as A49 — "Cross-Story Orthogonal-AC Invariants tested via Direct Artifact-Read"** (note: numbering will collide with A49 above; renumber at project-context update time).
**Impact**: pattern formalized for future N>1 epic invariants (e.g., when E14-S5 destructure list extends and E14-S1 allowlist must follow, the test catches drift at PR time).

### A52 — CSP env-var injection guard pattern
**Status**: project-context.md candidate addition
**Source**: Epic-14 boundary review patch P8
**Description**: Build-time env vars baked into security-relevant strings (CSP, CORS-allowed-origins, redirect-URI allowlists, Keycloak `${VAR}` substitutions) MUST be parsed via `new URL()` + emit only `.origin` (or equivalent normalization) — NEVER raw-concatenated. A malicious or mis-formatted CI variable can otherwise inject new directives. The patch added `normalizeOrigin()` + `normalizeHostSource()` to `frontend/src/lib/config/security-headers.ts` with 2 Vitest tests.
**Promote to project-context.md as A50** — "Build-time env vars in security-relevant strings MUST be parsed/normalized, not raw-concatenated".
**Impact**: prevents a class of security vulnerabilities where a compromised CI variable (or a fork operator mis-typing) can silently widen the security posture.

### A53 — When a SCP AC references an endpoint pattern, verify the endpoint exists at refresh time
**Status**: project-context.md candidate addition
**Source**: E14-S4 DEC-1 spec divergence
**Description**: SCP-2026-05-15 §5 E14-S4 named `10 req/min on /api/v1/auth/*`. IAB Connect has NO `/api/v1/auth/*` route (auth delegated to Keycloak). The DEC-1 surfaced 3 options; A=translate to IdentityEndpoints + UserEndpoints was chosen + documented. **The refresh surfaced the divergence**; a less-thorough refresh would have shipped a registered-but-never-fires named policy.
**Promote to project-context.md as A51** — "When a SCP/spec AC references an endpoint pattern (e.g., /api/v1/auth/*, /api/v1/admin/*), the `bmad-create-story` refresh MUST verify the endpoint exists in code and surface a DEC-Needed if it does not."
**Impact**: prevents register-but-never-fires policies + named handlers that never match any route.

## Significant discoveries

1. **E11-S2-D1 is deeper than A36 addresses.** The DEC-3=A attempt revealed that Infrastructure DI's Hangfire `UsePostgreSqlStorage(connectionString)` captures the connection string EAGERLY in the closure — A36's `AddInMemoryCollection` in ConfigureAppConfiguration runs AFTER the closure already baked-in the empty value. The proper fix is a lazy connection-string lookup OR a closure-deferred read. Tracked in deferred-work.md L344 + reinforced by this retro.
2. **CSP injection via env-var is a real attack surface for OSS forks.** A fork operator with a compromised GHA repo variable could ship a CSP that grants attacker-origin script-src permissions. P8 patched this at review time. A52 promotes the pattern to project-context.md so future security-relevant string interpolation goes through normalization.
3. **The `feedback_session_pacing_dev_cycles` memory was overridden by user explicit directive.** Memory recommends "commit-then-fresh for heavy dev cycles". User's *"ohne stopp"* + *"es handelt sich nicht mehr um einen mvp"* explicitly authorized the single-session approach. Outcome was successful (5 stories + review + retro in one session) but the session ran longer than typical. **Useful data point**: when user explicitly authorizes single-session bulk execution, the autonomous-mode escape + the dev-story workflow can deliver 5+ stories at ~2050-test green throughput.
