## 3. Challenges & lessons

- **A type-relocation AC collided with the import boundary (E23-S2 DEC-5).** AC-8 said "move the member DTOs/enums into the slice + re-export from `lib`". But the E21-S5 ESLint rule forbids `lib → features`, and `app/profile` (out-of-epic) + segments + duplicate components still import those types from `lib/api/members.ts`. The literal move was infeasible. Resolution: keep the canonical defs in `lib` (untouched), the slice `types/member.types.ts` re-exports them. → **A83**.
- **The shared client discards the response body the 409-synthesis needs (E23-S2 DEC-1).** `useApiClient` returns `{data,error,status}` and keeps only ProblemDetails `detail`/text — it drops `existingMemberId`, which REQ-018's 409 duplicate-synthesis requires. So `createMember`/`updateMember` had to stay raw-fetch (centralized in the slice api). → **A84**.
- **An unstable parent→child effect-callback clobbered just-set state (caught by S1).** `member-form`'s `onWatch` effect re-subscribed every render because the parent passed a fresh `handleWatch` closure each render → it cleared the just-set duplicate candidates → the new/Exact tests failed. Fixed with `useCallback` (+ a ref for the latest candidates). Same A64/A78 class, new seam. → **A85**.
- **"Destructive affordance" handling is contextual (A76 nuance).** S2's delete replaced a bare `confirm()` (no prior colour) → adopt the red `destructive` variant. S3's merge/dismiss already shipped an explicit **orange** affordance → preserving it was correct; converting to red would have been an unrequested visual regression. → **A86**.
- **Pre-existing lint errors stayed the next story's problem.** The 2 `set-state-in-effect` errors in the unmigrated `segments/page.tsx` were correctly scoped OUT of S2/S3 (A58/A72 changed-files gate) and dissolved naturally when S4's TanStack migration removed the manual fetch-in-effect. The changed-files gate kept scope honest.

## 5. Action items (promoted to project-context.md, A83–A87)

- **A83** — A type-RELOCATION AC must be validated against (a) the import-boundary direction (`lib ↛ features`) and (b) the COMPLETE consumer set incl. out-of-epic pages before treating it as mechanical. When either blocks the literal move, keep the canonical definition at the boundary-legal site and re-export it as the slice's surface, recorded as a DEC. [root: E23-S2 DEC-5]
- **A84** — Before migrating an endpoint to the shared `useApiClient` contract, verify the call site doesn't read response-body fields the client discards (it surfaces only ProblemDetails `detail`/text + status, dropping extras like a 409 `existingMemberId`). If it does, keep that endpoint on raw `fetch` inside the slice `api` module (URLs still centralized) and document it as a client exception. [root: E23-S2 DEC-1]
- **A85** (refines A64/A78) — a callback passed from a parent into a child effect that subscribes (e.g. RHF `watch`) MUST be referentially stable (`useCallback`, latest state via a ref), or the effect re-subscribes every render and can clobber just-set state. Stabilize parent→child effect-callbacks, not just mocked hooks.
- **A86** (refines A76/A80) — destructive-affordance handling is contextual: when an explicit colour already ships (e.g. an orange merge/dismiss confirm), PRESERVE it in a behaviour-preserving refactor; only introduce the `destructive` red variant where the prior surface had NO affordance (e.g. a bare `confirm()`). Converting an existing colour is an unrequested visual regression.
- **A87** (confirmed approach) — a green characterization net over the pre-refactor surface, wired with the forward-compat provider seam the target architecture will need (e.g. `QueryClientProvider`), is what makes safe parallel multi-agent extraction possible: the net being green IS the behaviour-preservation proof, so independent subagents can each own a surface and the orchestrator verifies centrally.
