## What went well

- **Reuse-first paid off.** The A56 spike mapped every shipped Communication pattern (`EmailCampaign` aggregate/repo/endpoint conventions, `IConsentRepository`, `IEmailSender`, the `VolunteerShiftReminderJob` shape, the `MarkReminderSentAsync` idempotency precedent, the campaigns list/detail/form UI). The automation vertical mirrored these instead of paralleling them — fast, consistent, low-surprise.
- **The S4 send-call isolation seam (S2 DEC-4) worked exactly as designed.** S2 isolated one private send method; S4 swapped it `IEmailSender`→`IMessageDispatcher` as a contained one-method change; S5 then layered the eligibility resolution in front of it — each a small, regression-guarded step. The seam discipline made the multi-channel slice additive.
- **Single-source criteria evaluator.** Relocating `SegmentCriteria`/`ApplyCriteria` from the Api layer to Application (`MemberSegmentCriteria`) gave the new resolver true reuse of the dynamic-segment logic — and a grep-tests-first check proved zero ripple before the move.
- **Honest QGT + Debug Logs surfaced the deferrals.** DEC-3's campaign-consolidation fallback and S3's detail-vs-list execution panel were disclosed in the Dev Agent Records, so the boundary review confirmed them as accepted rather than discovering them.

## What didn't (and what the boundary review caught)

- **Idempotency was not crash-safe (HIGH — P1).** S2's first cut recorded the whole execution (all idempotency keys) in one save *after* the send loop. A crash mid-batch commits nothing → the retry re-sends everyone. The "run-twice → one send" unit test was green because it never interrupts between send and persist. **Root lesson → A66.** Fixed: claim-before-send (persist `Pending` recipients first, then send + mark), + a claim-collision test + a claim-persisted-before-send test.
- **Time-relative triggers re-sent daily (HIGH — P2).** `EventUpcoming`/`MembershipRenewalDue` keys embedded `today+offset`; the shifting date produced a new key each day → daily email spam. **Root lesson → A67.** Fixed: date-free once-ever keys.
- **A disclosed "v1 simplification" was actually a defect.** The trigger evaluator's recipient-granularity was documented as a v1 limitation and the QGT marked AC-7 ✅, but the behavior (daily spam) was wrong, not merely reduced. **Root lesson → A68.** Fixed P2 makes it a harmless once-ever broadcast; the QGT row now states the concrete v1 behavior + the E5-FT-1 follow-up.
- **A Blind-Hunter HIGH was a false positive** (`FindFirst("sub")` → 401) — dismissed with evidence (`MapInboundClaims=false`). Healthy: the context-free layer correctly flagged a general risk; the project context resolved it.
