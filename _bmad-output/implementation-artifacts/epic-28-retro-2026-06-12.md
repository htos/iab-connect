## What went well

- **The S1 characterization net held as a true oracle through two very different migrations** — a client→RSC reshape (S2) AND an RHF+Zod reshape (S3) — catching nothing because nothing regressed, and the net-integrity auditor independently confirmed it was preserved, not weakened. The "pin behaviour first, then extract against the green net" discipline (A87) paid off again at the hardest epic.
- **The RSC island boundary landed clean on the first build.** No `onClick`/`useState` leaked into a Server Component; the testability constraint (no nested async — A105) was identified at design time, so `render(await Page())` worked uniformly and the build marked every page dynamic-RSC correctly.
- **The live-binding re-export (A106)** let the 3 form-page S1 specs stay 100% green UNCHANGED through the slice extraction — the cleanest possible "wrap" outcome for the form transports.
- **DEC discipline:** all DECs resolved to the recommended option (S2 DEC-1=B build-don't-wrap to avoid the EventDto widening; DEC-2..5=A; S3 DEC-1..5=A; S4 DEC-1..3=A), each recorded with A43 (a)/(b)/(c) Debug Log notes.

## What was hard / new (and how it was handled)

- **First break of the "net survives unchanged" invariant (A104).** Unlike E22-E27/E29, the 5 read-only specs' render harness HAD to change (client→RSC). The risk: a dev "adapts the tests to pass" instead of preserving behaviour. Mitigation: assert observable behaviour only, isolate the harness shell, and add a mandatory net-integrity auditor at the boundary. It returned ORACLE-INTACT — the only licensed deletion per spec was the loading-spinner test (A79).
- **RSC testability constraint (A105).** A naive `return <AsyncContent/>` from a tested route entry hands the test an unresolved promise. Solved by self-embedding fetch+getTranslations in each content component and flattening via re-export / `return ContentFn({id})`.
- **`useApiClient` is unusable on an anonymous surface** (401-gates) — the public reads use plain server `fetch`, the first slice in the program NOT built on `useApiClient`/TanStack. Confirmed by the A56 spike up front, so no rework.
- **The A79 loading-spinner delta** is a genuine behaviour change (client spinner → server-side wait). Accepted as the SEO/SSR improvement the epic exists for; documented per spec and in the boundary review.
