## What went well

- The pilot delivered the intended structure cleanly (feature slice, thin route, Radix dialog,
  Badge status, TanStack hooks) — a reusable recipe captured in the Pilot Result Note.
- The ATDD characterization net (S2) ran green before the refactor and stayed green, making the
  migration provably behaviour-preserving for everything it covered.
- Honest reality-correction: the dev-agent caught that the "semantic tokens already exist"
  Gate-1/AC premise was false (globals.css was 2 lines, no `@theme`/`:root`) and introduced
  the missing token layer instead of blindly marking the AC done.
- A-series discipline applied throughout: A64 (stable useTranslations mock), A58/A72
  (changed-files lint, prettier on new files only), A46 (cleanup), A56 (existing-impl spike).

## What was hard / lessons

- The characterization suite stayed green but did NOT catch **P2** (red→orange delete button)
  or **P3** (delete-error regression): it asserted text + API calls, not the visual/semantic
  variant nor the error/empty/loading lifecycle (no delete-fail case). Structural swaps
  (hand-rolled→library, manual fetch→TanStack) silently change exactly those.
- **P1** showed a token comment is not verification — the value was orange-500 while labelled
  orange-600, on the foundation token.
- TanStack adoption changed error/loading/retry semantics (retry:1, no spinner on refetch,
  sticky mutation error) that the test harness (retry:false) could not observe.

## Action items (promoted to project-context.md)

- **A76** — Characterization tests before a refactor must assert what the refactor is most
  likely to silently change — visual/semantic variant (e.g. destructive styling on a delete)
  AND error/empty/loading lifecycle incl. failure paths (delete-fails) — not just text + calls.
- **A77** — When introducing a design-token value, verify the hex/HSL against the named token;
  do not trust the comment. Especially for foundation tokens inherited by downstream epics.
- **A78** — Mock fidelity: when a consumed hook is memoized in prod and feeds an effect /
  query-key dependency chain, test mocks must return STABLE references (else refetch loop /
  prod divergence). Generalises A64 from `useTranslations` to any memoized hook.
- **A79** — Migrating manual fetch → TanStack Query: enumerate the semantic deltas (retry
  count, isLoading-on-refetch, error persistence) and decide per delta what to preserve; the
  retry:false test harness cannot see them.
